Checking and Updating TQSL's Configuration Data

TQSL is driven by Configuration Data that defines

  • Bands
  • Modes
  • Propagation Modes
  • Satellite Names
  • DXCC Entities
  • Primary Administrative Subdivisions (e.g. US States, Canadian Provinces, Russian Oblasts)
  • Secondary Administrative Subdivisions (e.g. US Counties)

Determining What Version of Configuration Data TQSL is Currently Using
Select the Help menu's About command; TQSL will display an About dialog that displays the Configuration Data version it's using.

Determining What Version of Configuration Data is Currently Available
The currently available Configuration Data version can be determined by visiting https://p1k.arrl.org/lotw/config_xml_version . The ARRL occasionally releases new versions of this Configuration Data, e.g. when a new DXCC entity is created, or when a new Satellite is launched.

Updating TQSL's Configuration Data
If TQSL is not using the currently available Configuration Data version, you can obtain the currently available version by following these steps:
  1. Log into the LoTW web page at https://p1k.arrl.org/lotwuser/default
  2. Click the Your Account tab
  3. in the Logbook Web Account menu on the left, click the Your Certificates item
  4. click the Download current TQSL Configuration file hyperlink; your web browser will display a File Download window
  5. in the File Download window, click the Open button; this will download the updated Configuration Data and invoke the TQSLCert application, which will display its TQSL Certificates window and a smaller Load Certificates dialog
  6. the last line in the Load Certificates dialog will indicate the status of the Configuration Data update; click the OK button to dismiss this dialog
  7. open the File menu in the TQSL Certificates window, and click the Exit item
