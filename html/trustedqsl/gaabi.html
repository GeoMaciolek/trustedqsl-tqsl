Trusted Gabbi (Global Amateur Interchange)  Formats

Discussion:


There are currently many different interchange formats that are being used by amateur radio today. By far the most popular isADIF (Amateur Data Interchange Format) for good reason. It is a very elegant, robust, and easy to read as raw data. It has many of the features found in XML without the need for special libraries and their associated learning curve. Having said that ADIF does have a short coming. It doesn't have any concept of record type. Discussions with the ADIF development reflector hasn't been fruitful. Therefore to meet the requires of this TrustedQSL/LoTW projects the standard must be forked. The new standard will be called Gabbi (Global Amateur Interchange) . It is hoped that the two standards will be merged at some point in the future. Therefore a design goal is for Gabbi format to be backward compatible with ADIF. This turns out to be fairly easy to do by declaring the QSO/Logbook record as the default record type.  The QSO record type will use the standard already defined by ADIF.

Requirements:

  • Record Type Field. This field will determine the record type
  • An algorithm for determining the signing order of the fields must conform to canonical parse rules.  
  • In addition the currently define QSO/Logbook record, QSL and Station Records must be defined.
  • Field definitions is only significant to that record.  For example a GRID_SQUARE field on a QSO would refer to the contact station grid square.  GRID_SQUARE on a station record would refer to the grid square of the station submitting the QSL
  • A new field or possibly as well define virtual field (ie: call and number of seconds since epoch) which can be common to all records and required for tCONTACT records will be the GUID (GlobalUniqueIDentifier).  This is in order that different records can be uniquely connected to other records in a portable and database independent way.
  • C/C++ functions that parse and create record strings.

 

New Terms:

tCONTACT - Trusted Contact Record.  This record contains on the information about the contact and a pointer to the tSTATION record.

tQSL - A Trusted QSL is a match set of a tSTATION record and a tCONTACT record.  Many tCONTACT records can reference a single tSTATION record. 

tSTATION - Trusted Station Record.  This record contains information about the station used to make contacts.

 

Record Types:

Record Name Description
QSO This is also the standard logbook format.  To maintain backward compatablity with ADIF this is the default record type.  For details of this QSO Record type seehttp:://www.hosenose.com/adif for details
tCONTACT

This record will contain conformation information to be sent to the party the user had a QSO with along with the STATION record.

tSTATION

This record will contain conformation information about the station that a QSO was made from. 

Common Fields:

These are field that is common to all record types.  These are fields common both in name and mean to more than one record type.

Name Type Required Description
REC_TYPE C NZ This defines the type of record.  The default is QSO record
SIGN C N This is the digital signature of this record.
GUID C N Global Unique IDentifier.  Exact description is TBD, but should be something like callsign and date/time group.  This maybe a viritual field.

tCONTACT (Trusted Contact) Record:

The tCONTACT record contains only the information that is unique to that contact.  Note that most of the fields are required.  This is because most of the fields are common to every contact.

Name Type Required Description
CALL C Y The call sign of station contacted.
CERT_GUID C Y Digital Certificate GUID
DTG H Y Date Time Group - 8 digit hex number of seconds since 1 Jan 1970 00:00:00
FREQ N Y in Megahertz.  If unknown use Frequency from ADIF table for the band.
MODE C Y SSB, CW, RTTY, TOR= AMTOR, PKT,AM,FM,SSTV,ATV,PAC=PACTOR,CLO=CLOVER
MY_CALL C Y My Call sign.
REC_TYPE C Y Always "tCONTACT".  This declares the record to be a Trusted Contact record.
RST_RCVD N Y Signal Report Received
RST_SENT N Y Signal Report Sent
SERIAL_NUM N N Serial number of the event.
SIGN C Y This is the digital signature of this record.  Base64 encoded
STATION_GUID C Y This is the Station Record GUID (pointer) for the station that was used to make this contact.
